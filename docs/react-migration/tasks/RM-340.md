## Scope

Create typed, tested React-facing persistence hooks for the four existing
browser behaviors without changing their keys or semantics:

- remembered member identity;
- reading scale;
- last-read Quran page;
- per-khatma du3a acknowledgement.

## Acceptance criteria

- Existing localStorage keys and fallback behavior remain compatible.
- Hooks are safe when storage is unavailable or contains invalid data.
- State changes persist without introducing shared UI state into Redux.
- Identity, reading scale, last page, and acknowledgement behavior have focused
  tests.
- Legacy UI and production entries remain untouched.
