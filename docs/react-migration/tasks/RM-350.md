## Scope

Create a reusable React test harness that renders components with the same
Redux, MUI, and hash-router provider contracts used by the React apps while
allowing tests to supply deterministic subscription data.

## Acceptance criteria

- Tests can render with Redux, MUI/RTL styling, and hash routing without
  repeating provider setup.
- Tests can provide deterministic roster, content, khatma, and assignment
  subscription sources without importing Firebase.
- The harness returns useful test controls, including the configured store and
  subscription publishers/cleanup behavior.
- Focused tests prove seeded rendering, subscription updates, routing, and
  isolation between renders.
- Production behavior, legacy UI, and production entries remain untouched.

## Ownership boundaries

- Keep the harness under `tests/`; production code changes require a concrete
  missing injection seam and must remain behavior-neutral.
- Do not add Firebase/emulator dependencies to component tests.
- Do not migrate feature screens or introduce new application state.
