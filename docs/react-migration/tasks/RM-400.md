## Scope

Replace the React member preview's unconditional shell with the member identity
gate. Use the existing Redux roster listener and compatible browser-persistence
hook so a member can choose, retain, and clear their trust-based identity without
adding authentication or changing legacy production behavior.

## Acceptance criteria

- With no remembered identity, the React member surface shows the centered gate
  without member navigation.
- The gate renders the live roster's loading, empty, ready, and error states with
  the existing Arabic strings.
- Choosing a member writes the existing `khatma.memberId` key and enters the
  member shell without starting duplicate subscriptions.
- Member feature code has a typed way to read the selected member and switch
  identity; switching clears persistence and returns to the gate.
- Listener cleanup remains owned by the shared provider and is proven on
  unmount; switching does not leak or duplicate listeners.
- Focused component/integration tests cover live roster changes, persistence,
  switching, error handling, navigation visibility, cleanup, and rerendering.
- Legacy UI and production entries remain untouched.

## Ownership boundaries

- Keep identity state browser-local; do not add it to Redux or Firestore.
- Reuse the global roster subscription already owned by `AppStoreProvider`.
- Expose a small member-identity contract that RM-420 can consume for the
  personal route; do not implement the personal insight screen here.
- Do not migrate member routes beyond the identity boundary or change the
  production entry.
