## Objective

Migrate the member khatma list and khatma landing routes to React/MUI while
preserving the baseline's relevant series selection, member assignment state,
group progress, warnings, history, navigation, completion write behavior, and
realtime/error handling.

## Acceptance

- `#/khatmas` renders one responsive card per active series, targets the khatma
  containing the member's pending chunk (or the latest), and shows title,
  percentage, progress, and the correct awaiting/done/pages line.
- `#/khatma/{id}` renders the ordered landing sections and correctly handles
  member-only warning severity, paused, awaiting, readable, pending completion,
  done, released-chunk, generic write-error, group-progress, pending-reader,
  history, loading, not-found, and subscription-error states.
- Routes use the existing typed identity context, Redux subscription bridge,
  domain functions, shared MUI primitives, hash router, and Arabic/RTL strings.
- Focused component/integration tests cover routing, realtime updates, listener
  cleanup, primary states and actions, and completion-write outcomes.
- Lint, focused/full tests, production build, bundle budgets, formatting/diff
  checks, and phone/desktop RTL browser QA pass before completion.

## Discovered context

- The legacy parity oracle is `src/ui/member/pages/khatmas.ts`.
- Khatma and per-khatma assignment entities already have provider-owned Redux
  listeners and selectors; active subscriptions must be acquired by React route
  components rather than by feature-local Firestore listeners.
- `activeSeriesGroups`, `completedInSeries`, `seriesTitle`, `khatmaProgress`,
  pending-chunk helpers, and `warningLevel` supply the existing domain behavior.
- `markRoundDone` and `ReleasedChunkError` remain the completion write boundary.
- The React member shell already recognizes `khatmas`, `khatma`, and
  `khatmaRead`; RM-410 supplies the first two route bodies only.

## Implementation

- Replaced the member preview content for `khatmas` and `khatma` with React/MUI
  list and landing pages while retaining the shared route marker contract.
- Added a persistent member-experience reconciler that retains exactly the
  selected member's active-khatma assignment subscriptions through the existing
  reference-counted Redux bridge and cleans them up as membership changes.
- Reused the established series/progress/warning selectors and shared surface,
  progress, status, action, and feedback primitives for all route states.
- Kept completion behind the injectable `markRoundDone` operation, with pending
  disablement, immediate success feedback, and distinct released/generic errors.
- Preserved the documented transient empty-list quirk and legacy loading,
  membership/not-found, warning privacy, paused, awaiting, done, history, and
  pending-reader behavior.
