## Outcome / scope

Migrate the member personal and settings routes into the React preview. The
personal route must show the selected member's live identity and lifetime Quran
reading insight. The settings route must expose the compatible reading-scale
control, apply it live, and remember it through the existing browser-persistence
contract.

## Acceptance criteria

- `#/personal` renders the Arabic heading, greeting, selected member name,
  lifetime completed-page count, percentage of 604 pages, and progress.
- Personal insight updates from live member data and preserves the established
  zero/empty behavior while roster data is unavailable.
- The existing switch-person action remains reachable and clears the compatible
  remembered identity through the identity context.
- `#/settings` renders the Arabic settings heading, a 1–5 reading-scale slider,
  and sample text matching the legacy behavior.
- Reading-scale changes apply live to the member experience and persist through
  `khatma.readingScale`; remembered values are restored on remount/reload.
- Member navigation reaches both routes and route-local interactions remain
  stable across ordinary React rerenders.
- Focused component/integration tests cover personal insight, live updates,
  settings persistence/restoration, scale application, and navigation.
- Legacy UI, production entries, dependencies, lockfiles, Firebase data/rules,
  and unrelated member routes remain untouched.

## Files or ownership boundaries

- Prefer route components under `src/app/member/` and reuse existing shared
  navigation, feedback, progress, and persistence contracts.
- Keep reading scale browser-local; do not move it into Redux or Firestore.
- Reuse the global roster subscription and selected-member identity contract;
  do not add feature-local listeners.
- Do not migrate reader, completion, admin, legacy production, or unrelated
  member behavior.

## Decisions and risks

- The legacy personal percentage is based on `lifetimePercent(..., 604)`; verify
  the React domain/selector shape before composing it.
- The legacy settings control survives imperative rerenders. In React, preserve
  equivalent interaction state through component ownership rather than copying
  the native `<details>` implementation literally.
- Reading scale must affect the intended member reading surface without
  expanding RM-420 into the Quran reader migration.
