## Outcome / scope

Migrated the route-independent member completion/du3a interrupt to React. The
boundary preserves assignment subscriptions while suppressing the member shell
and route content until each eligible khatma is acknowledged.

## Acceptance criteria

- Required reciter variants match: unset/self reciters see the du3a and other
  members see the designated reciter's live roster name.
- Live `content.du3aText` is used with `DEFAULT_DU3A_TEXT` as the null fallback.
- Acknowledgement uses the exact existing `khatma.du3aAck.{khatmaId}` browser
  key and returns to the unchanged route after the final pending completion.
- Only active, completed khatmas belonging to the selected member interrupt;
  multiple unacknowledged completions are presented in source order.
- Member navigation and route content are unmounted for the full interrupt.
- Focused automated coverage and mobile/desktop RTL browser QA pass.

## Decisions and risks

- Reused `khatmaProgress`, Redux's live roster/content/assignment state, and
  `useDu3aAcknowledgement`; no duplicate state, listener, or persistence layer
  was added.
- Kept `MemberAssignmentsSubscriptions` outside the interrupt so realtime
  completion state remains live while the member shell and any reader unmount.
- Used one acknowledgement candidate component per completed khatma so hooks
  remain stable and already-acknowledged candidates advance to the next one.
- Browser QA caught MUI typography/flex cascade differences; the final screen
  uses the global Quran text class on a plain paragraph and a compact centered
  Done action, matching the legacy presentation.
