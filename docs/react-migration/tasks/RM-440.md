## Outcome / scope

replacing the previous preview fallback in `MemberRouteContent`:

- Browse reader (`#/quran`, `#/quran/{page}`) — free reading over all 604 pages

## Decisions and risks

- Browse reader derives its page from the URL (last-read fallback) — no dual
  source of truth, and navigation only pushes the hash. The assigned reader keeps
  index state in a component keyed `{khatmaId}:{round}`, so realtime ticks never
  reset scroll/page while a new round remounts fresh.
- Per-page fetch uses an effect whose cleanup flag plus a settled-page match are
  the stale-navigation guard (a newer page always wins) and keep setState out of
  the synchronous effect body (React 19 lint).
- The assigned finish footer mirrors the legacy reader exactly: any failure
  surfaces `member.saveError` (no `ReleasedChunkError` special-casing, which
  lives only on the landing page).
- Jump selects are placeholder-style momentary triggers (they do not track a
  value), matching the legacy jump-only intent; the page input commits on
  blur/Enter and remounts on committed-page change to resync without an effect.
