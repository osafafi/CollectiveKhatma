## Refresh design intent

- Warm, low-glare paper palette (reduced pure-white glare for long reading).
- Calm emerald "Quran green" primary + antique-gold accent; warn/accent split
  (theme-map R3) so warnings and the chart "pending" segment are distinguishable.
- Higher text contrast and roomier line-height for senior readability (WCAG AA+).
- Softer, gentler card elevation and radii — modern but calm, not "techy".
- A visible keyboard `:focus-visible` ring (accessibility improvement).
- Legacy Tailwind `theme.css` is intentionally left on the old palette (it is
  production until RM-600 cutover and is deleted at RM-620); the React MUI theme
  now diverges. The `mui-theme` sync guard is updated to encode this intentional
  divergence instead of asserting equality.

## Intentional deltas (approved under OD-03)

1. **Refreshed palette** (table above) — warm low-glare paper, calm emerald
   primary, distinct gold accent, higher-contrast ink/muted. Differs from the
   legacy Tailwind look by design.
2. **Accent/warn split (theme-map R3 resolved)** — accent is now a gold (hue
   ~42°) clearly separated from the amber warn (~26°) and red danger (~5°); they
   were the same `#b45309` before.
3. **Gold accent uses dark ink text**, not white — the one filled semantic color
   that does (legibility on a light gold).
4. **Softer card radius (18) and warm layered shadow** — gentle modern depth vs
   the flat 1px hairline.
5. **Roomier UI line-heights** for senior reading comfort (sizes unchanged, so no
   layout shift).
6. **Visible keyboard `:focus-visible` ring** — an accessibility improvement over
   the baseline (which had no explicit focus ring). Recorded per inventory §1.7 /
   §5 guidance not to slip improvements in as "parity".
7. **React ≠ legacy palette during transition** — `theme.css` intentionally kept
   on the old palette (production until RM-600; deleted at RM-620); the theme
   sync test now guards the divergence instead of equality.

No functional, data, domain, routing, listener, or reader-lifetime behavior was
changed; the refresh is entirely theme-level. P1 (reader survives ticks) and the
realtime paths are unaffected and remain covered by RM-440/RM-450.
