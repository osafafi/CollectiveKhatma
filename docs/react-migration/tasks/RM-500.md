## Approach

- `AdminApp` → `AppProviders` → `AdminExperience` = `AdminAssignmentsSubscriptions`
  (persistent, route-aware) + `AdminShell` (persistent header + nav) +
  `AdminRouteContent` (switch on `useAdminRoute()`).
- Distribution reuses the injectable data boundary via
  `useWriteOperation('runDistribution')`; confirmation via `useConfirmation()`;
  the surah/juz maps distribution needs are loaded through a small
  `useQuranScopeMaps()` hook (React twin of the legacy admin boot load of
  `getQuranIndex()` → `buildPageUnitMaps`).
- New `src/app/admin/todayIso.ts` — React-owned local-calendar "today" so the
  React tree does not import the legacy `src/ui` layer (deleted at RM-620).

## Intentional deltas (recorded per inventory §1.7 / §5)

1. **Distribution status a11y** — the legacy renders every distribute status,
   successes and errors alike, in the same green `text-success`. The React version
   keeps successes on a success tone with `role="status"` but shows the
   `alreadyDistributed` / `distributeError` outcomes on an error tone with
   `role="alert"`, so a failed distribution is no longer announced as success.
   An improvement recorded per §1.7, not hidden as parity.
2. **Single `h1` per route** — the persistent admin title is a non-heading label
   (the legacy renders it as an `h1` in addition to each page's own `h1`). Each
   admin route now has exactly one `h1` (its page heading); the visible title text
   and its persistence are unchanged.
3. **Sub-block separation** — khatma sub-blocks are separated by `Stack` spacing
   inside an outlined surface rather than the legacy hairline bottom-borders — the
   modern-but-calm surface treatment already approved under OD-03.
4. **Inherits the OD-03 theme refresh** automatically via the shared MUI theme —
   no new decision (per NOW.md handoff).

No functional, data, domain, routing, or distribution-algorithm behavior was
changed; the parity-critical behaviors (P7/P8/P9) are reproduced.
