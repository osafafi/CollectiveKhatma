## Scope

Third Phase 5 task. Migrate the admin **Khatmas** route (`#/khatmas`) into the
RM-500 admin shell: the list of all khatmas first, then the gated draft-heavy
create form (scope, member picker, per-member capacity, reciter, backfill,
series continuation).

Parity oracle:
[`REACT_MIGRATION_UI_INVENTORY.md`](../../../REACT_MIGRATION_UI_INVENTORY.md)
§3.3 (Khatmas list + create), §4 **P2** (draft survival), §5 quirk 4 (unused
strings — do not resurrect).
Legacy source of truth:
[`src/ui/admin/pages/khatmas.ts`](../../../src/ui/admin/pages/khatmas.ts).

In scope:

- **List:** all khatmas, active first then completed, each by `createdAt` desc;
  each line: title, status badge (`statusActive`/`statusCompleted`), percent
  (completed = 100) → link to `#/khatmas/{id}`. Empty → `noActive`.
- **Create gate:** `createNewButton` reveals the form; `cancel` hides it.
- **Create form:** series name + continuation note (`findSeriesByName` →
  `continuesSeries N`); scope select `full`/`range`(from/to)/`surahs`
  (surah-name checklist, loading until surahs load); member picker checkboxes
  (disabled members labeled, empty roster → `emptyRoster`); per-member additive
  capacity (pages + surah-name select + juz; solo ⇒ 1 juz, group ⇒ roster
  `pagesPerDay` pages); reciter select (`reciterAuto` else a member); backfill
  (created date + series-number override); `createButton` → `createKhatma`.
- **Validation/errors:** `seriesNameRequired`, `selectMembers`, `createError`.
- **Series logic:** matching name continues the series with the next number;
  new name ⇒ new `seriesId`, number 1; a valid override wins.
- **Draft:** the whole create form is draft-held and survives unrelated snapshots
  (**P2**); reset on successful create.

Out of scope: Khatma detail (RM-530, sibling task in this session), Settings
(RM-540), the form-draft stability proof (RM-550), and the full admin
integration suite (RM-560). RM-520 ships focused Khatmas list/create tests.

## Approach

- New `src/app/admin/pages/KhatmasPage.tsx` (`AdminKhatmasPage`); wire it into
  `AdminApp`'s `AdminRouteContent` (`khatmas` → it).
- New `src/app/admin/useSurahs.ts` hook (loads `getSurahs()` once, `null` until
  ready) for the surah-name checklist + capacity selects — the twin of the
  legacy admin boot `getSurahs()` load. `useQuranScopeMaps` supplies
  `surahToPages` for `resolvePageScope`.
- Reuse shared primitives: `SurfaceCard`, `AppTextField`, `AppSelectField`,
  `AppCheckboxField`, `AppButton`, `StatusChip`, `DonutChart`-free list rows.
- **Cross-page prefill (shared with RM-530):** a new
  `CreateKhatmaPrefillProvider` at `AdminExperience` level holds a one-shot
  prefill payload so the detail page's `startNext` can seed this create form
  across the navigation unmount. The live editing state stays lifted in
  `AdminKhatmasPage` (survives the show/hide toggle and unrelated snapshots, P2);
  it is route-scoped, the intentional delta RM-510 established and RM-550
  formalizes.

## Acceptance criteria

- List orders active-first then `createdAt` desc, links to detail, shows correct
  status badge + percent; empty → `noActive`.
- Create form gated behind `createNewButton`; cancel hides without losing the
  entered draft; the full form (scope/members/capacity/reciter/backfill) drives
  `createKhatma` with the correct series id/number, capacities, and createdAt.
- Continuation note appears when the typed name matches an existing series.
- Validation covers blank name, no members, and scope-resolution failure.
- Draft survives unrelated snapshots (P2).
- Typecheck, lint, and the full test suite pass (Khatmas tests added).

## Intentional deltas (recorded per inventory §1.7 / §5)

1. **Draft scope** — the create form draft is lifted into `AdminKhatmasPage`
   (route-scoped) rather than an app-lifetime `AdminDraft`; it survives unrelated
   snapshots and the show/hide toggle. RM-550 formalizes the cross-form model.
2. **Create error a11y** — `createError`/`seriesNameRequired`/`selectMembers`
   carry `role="alert"` + error tone (the legacy shows plain danger text). No
   copy or trigger change.
3. **Cross-page prefill mechanism** — `startNext` (RM-530) hands off through a
   one-shot React context (`CreateKhatmaPrefill`, peek-then-clear-in-effect so a
   StrictMode double render never double-consumes) instead of mutating a shared
   draft object; the visible behavior (jump to `#/khatmas` with a prefilled,
   opened create form) is identical.

## Implementation

- New `src/app/admin/pages/KhatmasPage.tsx` (`AdminKhatmasPage`):
  - `KhatmasList` sorts active-before-completed then `createdAt` desc; each
    `KhatmaListLine` reads its own assignments and links to `#/khatmas/{id}` with
    a status `StatusChip` and percent (completed = 100). Empty → `noActive`.
  - `CreateArea` holds the whole create form in one lifted `draft` state so it
    survives snapshots (**P2**) and the show/hide toggle; cancel hides without
    resetting, and only a successful create resets + closes.
  - Scope select (`full`/`range`/`surahs`), a member picker (`AppCheckboxField`,
    disabled members labeled), per-member capacity rows (solo ⇒ 1 juz, group ⇒
    roster pace, locked at selection like the legacy `??=`), reciter select,
    backfill date + number, and the continuation note via `findSeriesByName` /
    `nextSeriesNumber`. `onCreate` mirrors the legacy: validation
    (`seriesNameRequired` / `selectMembers` / `createError`), `safeUUID` for a new
    series, override-wins numbering, `pickDuaReciter` fallback, and `createKhatma`.
- New `src/app/admin/useSurahs.ts` (surah names, `null` until loaded) and
  `src/app/admin/SurahCapacitySelect.tsx` (shared with RM-530).
- New `src/app/admin/createKhatmaPrefillContext.ts` (context + hook) and
  `CreateKhatmaPrefill.tsx` (provider), wired into `AdminExperience`.
- `AdminApp.tsx` routes `khatmas` → `AdminKhatmasPage`.
- Tests: `tests/app/admin-khatmas.test.tsx` (6) — ordering/badge/percent/link,
  empty, create gate + draft-survives-cancel, blank/no-member validation,
  full-scope create (solo capacity default, reset), and series continuation.
