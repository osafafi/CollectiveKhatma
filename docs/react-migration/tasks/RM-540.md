al-khatma editor plus the shared 1–5 reading-scale control.

In scope:

- **Reading control**: the shared `<details>` popover with the 1–5 font-size

## Approach

- New shared `src/components/primitives/ReadingScaleControl.tsx`: the `<details>`
  disclosure (slider + sample) extracted **verbatim** from the member
  `SettingsPage` so both apps render one control (**P11**). The member page is
  refactored to consume it — no behavior change (its RM-420 tests still pass).
- New `src/app/admin/pages/SettingsPage.tsx` (`AdminSettingsPage`): heading +
  `Du3aEditor` (SurfaceCard) + `ReadingScaleControl`.
  - **Du3a P3:** the textarea is component-local state seeded from
    `selectContent().du3aText`; a `touchedRef` freezes seeding once the admin
    edits, and a successful `setDu3aText` clears it (mirror of the legacy
    `du3aTouched`). The Arabic textarea keeps the `quran-text` reading font.
  - `save` uses `useWriteOperation('setDu3aText')`; status derives from the
    operation state.
- `AdminApp`'s `AdminRouteContent` lifts `useReadingScale()` + `settingsOpen`
  (like the member `MemberRouteContent`) so the applied scale and the disclosure
  open/close state persist across route navigation. The dead route placeholder
  (`AdminRoutePlaceholder`, `PENDING_ROUTE_HEADING`) is removed — every admin
  route is now migrated.

## Acceptance criteria

- Du3a editor seeds from live content, saves through `setDu3aText`, and shows
  `saved`; a failure shows an error-tone `alert` (see delta 1).
- Once edited, an unrelated content snapshot does not overwrite the draft (P3);
  before editing, snapshots keep seeding it.
- Reading scale restores from localStorage, live-applies to
  `<html data-reading-scale>`, and persists; one control shared with the member
  app (P11).
- Disclosure open state survives route navigation.
- Typecheck, lint, and the full test suite pass (settings tests added).

## Intentional deltas (recorded per inventory §1.7 / §5)

1. **Save a11y/status** — the `saved` note carries `role="status"`; a save
   failure surfaces an error-tone `alert` (the legacy settings save has no error
   surface and shows only the green note). Behavior otherwise matches; this
   mirrors the RM-500/RM-530 status deltas.
2. **Draft scope** — the du3a draft is component-local/route-scoped and survives
   unrelated snapshots via the P3 touched guard (the RM-510 draft-scope delta).
