## Observation (shared, not an admin parity gap)

Admin route headings (`<Typography component="h1" variant="h2"
color="primary.main">`) render in ink `#26312b` rather than emerald: the
`color="primary.main"` prop produces no color rule on the h2-variant heading. This
is **identical in the member app** (the member gate/route headings render the same
ink and were accepted in RM-460), is highly legible (ink-on-bg 11.98:1), and falls
under the OD-03 approved-refresh umbrella. It is therefore a shared centralized-
theme detail consistent across both apps — recorded here as a candidate for a
future shared-theme touch-up if emerald headings are desired, **not** an admin
regression or a Phase 5 blocker.
