# RM-620 — Remove legacy UI and Tailwind

- Status: DONE
- Owner: Joint (this pass driven by Codex)
- Dependencies: RM-610 (complete)
- Baseline commit: `cf11a52` (RM-600 + RM-610 cutover)

## Scope

Delete the obsolete framework-free member/admin bootstraps and DOM renderers now
that both production entries mount React. Remove Tailwind's CSS import, Vite
plugin, packages, and lockfile graph. Keep framework-independent data/domain
contracts and React-owned assets, including the Quran font, static icon
overrides, and the MUI theme.

## Acceptance criteria

- `src/member.ts`, `src/admin.ts`, and `src/ui/**` are removed.
- Tailwind is absent from CSS, Vite configuration, direct dependencies, and the
  lockfile.
- Retained React CSS/assets have no dependency on the deleted legacy UI.
- Tests and comments no longer import or link to deleted source.
- Dead-import search, typecheck, lint, full tests, production build, and bundle
  budgets pass.

## Decisions and risks

- `src/data/**`, `src/domain/**`, `src/content/**`, and React application code are
  outside the legacy deletion boundary even where the removed UI consumed them.
- `src/theme/theme.css` was entirely the legacy/Tailwind copy; its retained rules
  already existed independently in `globalStyles.ts`, so the file was deleted.
- Historical task records may continue to name deleted parity-oracle files, but
  executable imports and current architecture/developer documentation must not
  encode the retired UI.
