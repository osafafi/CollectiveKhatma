## Scope

Exercise the production React data flow with two independent clients connected
to the local Firestore emulator. Verify realtime roster, khatma, assignment,
distribution, completion, reload/reconnect, and subscription cleanup behavior.

## Acceptance criteria

- A write from one client is observed by the other through realtime listeners.
- Distribution updates the active khatma and assignments consistently in both
  clients, and completion propagates without a reload.
- Reload/reconnect restores the current emulator state without duplicate writes.
- Unsubscribed listeners stop receiving updates, assignment state is removed,
  and global listeners return to idle.
- Focused emulator coverage and the normal typecheck/lint/test gates pass.

## Decisions and risks

- Validation is emulator-only; this task does not authorize staging/live access
  or deployment.
- Use isolated data and deterministic dates so the validation can be repeated.
- Prefer an automated two-client harness for durable evidence, supplemented by
  a browser smoke pass where it covers behavior not visible at the data layer.
