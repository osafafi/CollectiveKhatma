# RM-650 — Accessibility, RTL, responsive QA

- Status: DONE
- Owner: Joint (this pass driven by Codex)
- Dependencies: RM-620 (complete)
- Baseline commit: `ae7546a` (RM-620 + RM-630 completion)

## Scope

Validate the member and hidden admin React surfaces at mobile and desktop
widths for keyboard operation, focus behavior, labels, contrast, RTL direction,
portal direction, touch sizing, and layout integrity. Fix regressions found and
add focused automated coverage for durable contracts.

## Acceptance criteria

- Primary member and admin flows are usable by keyboard with visible, logical
  focus and correctly restored dialog focus.
- Controls and status content have useful accessible names and semantics.
- Document, theme, and portaled UI remain RTL; visual order and text alignment
  are appropriate for Arabic content.
- Mobile and desktop layouts avoid clipping, overlap, and horizontal overflow;
  interactive targets meet the established sizing rules.
- Representative contrast checks and focused accessibility/layout tests pass.

## Decisions and risks

- Keep the hidden admin entry hidden and test it only at `admin-nano.html`.
- Prior browser QA plus current automated contracts cover representative routes
  and reusable primitives without encoding brittle pixel snapshots.
- Any dependency or entry change must retain the production bundle budgets.

## Findings and fixes

- The 55%-alpha emerald focus ring measured only 2.33:1–2.43:1 against the two
  app backgrounds. Raising it to 70% produces 3.06:1–3.22:1 and meets the 3:1
  focus-indicator floor.
- Success and warning text on their 10%-tinted status surfaces measured
  3.91:1–4.33:1. Dedicated dark semantic text tones now produce
  5.42:1–5.97:1 while the established filled colors remain unchanged.
- Responsive shell/nav values now live in one implementation-owned contract:
  full-width 56px-minimum mobile tabs with 112px content clearance, switching at
  `lg` to a 96px full-height RTL right rail with 32px content bottom padding.

## Verification

- Focused automated QA: 33 tests pass across navigation, shared primitives,
  theme, and RTL provider behavior. New coverage proves logical keyboard tab
  order, portalled-dialog RTL styling, confirm autofocus, Escape close/focus
  restoration, responsive breakpoint contracts, and calculated contrast floors.
- Existing page/integration suites continue to address labels, semantics,
  alerts/status, form focus stability, and both member/admin route journeys.
- Live emulator-backed evidence from RM-460 and RM-570 remains applicable to the
  unchanged React layouts: member and hidden admin surfaces were each walked at
  375px mobile and 1280px desktop, confirming the bottom bar/right rail, RTL
  placement, route content, and live controls. RM-620 only removed the retired
  UI while retaining React global styles; RM-630 only changed chunking.
- A fresh in-app-browser rerun was attempted after seeding the local emulator,
  but the browser surface hard-blocked localhost navigation after its initial
  pre-server connection failure. Its policy prohibited switching browser
  surfaces as a workaround, so no new visual observation is claimed here.
- Source review found fixed field widths bounded within wrapping rows, flexible
  content columns with `minWidth: 0`, wrapped action groups, and no unbounded
  layout width that would introduce horizontal overflow at the target sizes.
- Full suite: 39 files passed, 1 skipped; 225 tests passed, 1 skipped. Lint,
  typecheck/build, diff whitespace, and production bundle budgets pass.
- Final production budgets: member 341.59 kB JS gzip / 387.75 kB transfer;
  admin 344.72 kB / 390.98 kB, within the fixed 350/400 and 375/425 kB limits.
