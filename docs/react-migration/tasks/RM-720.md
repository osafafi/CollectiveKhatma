## Review evidence

### Boundaries and state ownership

- Static import review plus the ESLint guardrails confirm every `firebase/*`
  import is under `src/data`; `src/domain` imports only domain modules.
- Global roster/content/khatma snapshots and per-khatma assignments are the only
  Redux-owned shared records. Listener errors are stored as strings; SDK objects,
  snapshots, unsubscribe functions, DOM/MUI objects, and local form/reader state
  remain outside Redux.
- Feature mutations execute through `useWriteOperation` and the injectable
  `WriteOperations` adapter. Feature imports from `src/data` are limited to
  specific result/error contracts used for user feedback; no page starts a read,
  write, or listener directly.
- Browser storage access is centralized in `browserPersistence.ts`; routing uses
  the shared typed hash contract.

### Subscription lifecycle

- `AppStoreProvider` owns one bridge per store/source pair and retains the three
  global listeners in an effect.
- The bridge guards late callbacks, surfaces synchronous/asynchronous errors,
  reference-counts assignment listeners, makes releases idempotent, reverses
  global cleanup, dispatches stopped state even when unsubscribe throws, and
  removes assignment state on final release.
- Persistent member/admin subscription components select only the assignments
  each surface needs. Focused tests cover sharing, Strict Mode-style cycles,
  route/identity reconciliation, errors, remote ticks, and cleanup; RM-640 adds
  the two-client emulator proof.
